#RF

  - [ ] O usuario vai poder criar uma nova transação
  - [ ] O usuario vai poder obter um resumo da sua transação
  - [ ] O usuário deve poder listar todas as transações feitas.
  - [ ] O usuário deve poder ver uma transação unica.

#RN

 - [ ] A transação pode ser do tipo crédito que somará ao valor total ou do tipo débito que subtrairá. 
 - [ ] Deve ser possivel identificarmos o usuário entre as requisições. 
 - [ ] O usuário deve ver transação que ele criou.
